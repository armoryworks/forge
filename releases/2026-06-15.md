# Week of June 15, 2026

Detailed notes for everything that shipped this week. The short version is in
RELEASES.md at the top of this repo.

Versions released this week: API 0.1.0 then 0.1.2, Web 0.1.0, and the database
schema moved to forge-db.


## Database (forge-db, shipped in API 0.1.2)

Entity Framework migrations were removed. The schema is now plain SQL in the
forge-db project, one file per object, assembled into a single schema file that
the API ships and applies.

Why: the schema now has one owner. The old migration setup could not describe a
few things that matter, so they were patched in by hand. Those now live in
forge-db with everything else:

- the vector search extension used for document search, and
- the database triggers that prevent posted accounting entries from being
  edited or deleted.

What happens on deploy: when the API starts it checks whether the database
already has the schema. If it does, it does nothing. If the database is empty it
creates the whole schema in one step. Every server already had the schema, so
this release changed no data anywhere.

The forge-db project also merged its first version this week: the schema tree
plus the tooling that assembles it and checks it against the source.


## Web app and API (0.1.0)

These are the fixes from the issue list.

Part search on the job edit screen (issue 28). Typing a search did nothing. The
parts list had changed to a paged response and the job screen was still expecting
a plain list, so it hit an error that was never shown on screen. Fixed by reading
the paged format. Checked in a browser against a live server.

Price prefill on order lines (issue 26). When you pick a part on a quote line or
a sales order line, the unit price now fills in from that customer's price list.

Draft sales order header (issue 8). After turning a quote into an order, the PO
number, credit terms, and requested delivery date could not be set. The draft
order now has an edit form for those three fields.

Estimate to quote line items (issue 24). Converting an estimate to a quote was
producing an empty quote. It now copies the line items across. If the estimate
was only a single total with no lines, that total comes across as one lump sum
line that can be itemized later.

Sales order visibility (issue 25). A sales order created from an accepted quote
was not appearing in the Sales Orders list, because that list only showed orders
already in production. Draft orders now appear in the list, so a new order is not
lost before someone confirms it.

Job to sales order line (issue 27). When creating a job you can now attach it to
an open sales order line. The picker shows only lines with no open job by
default, and has a checkbox to show lines that are already assigned.


## Release and deploy

The version moved from the 0.0.x line to 0.1.0. This marks the database moving to
forge-db and the accounting suite reaching feature complete. The accounting
features stay off until go-live.

0.1.0 went to all three installs: the armoryworks API server, the armoryworks web
server, and the Armory Plastics server. 0.1.2, the database change above, went to
the two servers that run the API. Each deploy keeps the previous version ready to
roll back to.
